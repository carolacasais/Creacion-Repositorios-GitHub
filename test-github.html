Guía para crear repositorios en GitHub desde VSCode y actualizarlos mediante commits.

Creación de repositorios en Github mediante Visual Studio Code

En primer lugar debemos tener una cuenta en GitHub, Git instalado en nuestro ordenador así como VSCode (con la extensión "GitHub Pull Requests and Issues")

  1. Crear un documento con VSCOde en una carpeta (ubicada en d:)
  2. Abrir PowerShell y localizar la carpeta
    • d:
    • cd + el nombre de la carpeta
    • ahi ya estamos dentro de la carpeta, escribimos "code ." para abrirla en VSCode y crear el archivo .git
  3. Iniciamos sesión en GitHub desde el icono de VSCode
  4. Vamos al panel de comandos (View > Command Palette )
  5. Ahí escribimos "> Publish to GitHub"
  6. A continuación escribimos el nombre con el que queremos publicar el repositorio y si queremos hacerlo de forma pública o privada.

NOTA:

  • Git PUSH: Enviar los cambios que has hecho de forma local al repositorio remoto
  • Git PULL: Recibir los cambios que hay en el repositorio remoto a tu repositorio local

Actualización de repositorios en GitHub mediante commits desde Visual Studio Code

Cuando se realizan modificaciones en un archivo se notifica en la pestaña "Source Control de VSCode. Para trasladarlas a nuestro repositorio de GitHub debemos seguir los siguientes pasos

  1. Vamos a la pestaña "Source Control" y hacemos click en el signo "+" de los archivos que queramos actualizar.
  2. Escribimos el nombre del commit (ej.: Primer commit)
